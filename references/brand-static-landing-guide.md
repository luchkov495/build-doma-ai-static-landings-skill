# DOMA.ai: спецификация для генерации статичных лендингов

Дата анализа: 2026-04-27  
Задача документа: дать AI-генератору достаточно точные правила, чтобы делать новые статичные продуктовые лендинги DOMA.ai в визуальном и текстовом стиле оригинальных страниц.

## 1. Источники и границы анализа

Проанализированы страницы:

- https://doma.ai/client_center
- https://doma.ai/count_center
- https://doma.ai/strahovanie-zhkkh

Сняты desktop и mobile скриншоты, браузерные computed styles, видимая структура блоков, повторяющиеся паттерны текста, CTA и визуальные токены.

Локальные референсы:

- `doma-client-center-desktop.png`

## 2. Суть бренда и продукта

DOMA.ai говорит как практичный B2B-сервис для управляющих организаций в ЖКХ. Это не эмоциональный стартап-лендинг и не корпоративная витрина. Основной образ: спокойная технологичная платформа, которая берет на себя операционные задачи УК/ТСЖ и снижает расходы.

Ключевая аудитория:

- управляющие компании;
- ТСЖ, ЖСК, ТСН;
- крупные холдинги в ЖКХ;
- иногда РСО и смежные организации, если продукт связан с начислениями/сервисом.

Главные мотивы:

- снизить расходы;
- делегировать операционную работу;
- улучшить отношения с жителями;
- повысить качество сервиса;
- не зависеть от больничных, отпусков и ручных процессов;
- получить прозрачную стоимость;
- сохранить контроль и отчетность.

Запрещенный тон:

- хайп, громкие обещания без основания;
- "революционный", "лучший на рынке", "уникальный" без доказательств;
- B2C-эмоциональность;
- абстрактные фразы вроде "цифровая трансформация нового поколения".

Нужный тон:

- уверенный, спокойный, прикладной;
- конкретный: заявки, начисления, квитанции, лицевые счета, ГИС ЖКХ, МКД, жители;
- выгода объясняется через операционные процессы, деньги, время, доверие.

## 3. Общая визуальная система

### 3.1 Палитра

Использовать темную бирюзово-графитовую основу и яркий зеленый CTA.

CSS-переменные:

```css
:root {
  --doma-bg: #012025;
  --doma-card: #17313B;
  --doma-card-deep: #10252E;
  --doma-card-soft: #335866;
  --doma-green-glow: #26C756;
  --doma-blue-glow: #4BA2E4;
  --doma-green: #2BC359;
  --doma-green-active: #00B538;
  --doma-green-hover: #42D471;
  --doma-white: #FFFFFF;
  --doma-light: #F2F4F6;
  --doma-text-dark: #222222;
  --doma-text-ink: #1D1D1F;
  --doma-text-muted: #738290;
  --doma-text-muted-2: #95A2B0;
  --doma-text-on-dark: #D0D3E5;
  --doma-border-muted: #707695;
  --doma-warning: #FF8562;
}
```

Правила применения:

- Основной фон страницы: `#012025`.
- Большие темные карточки: `#17313B`.
- Более глубокие вложенные темные элементы: `#10252E`.
- Зеленый используется только для действий, активных табов, чеков и небольших акцентов.
- Белые секции нужны для доказательств, отзывов и экосистемы продуктов.
- Нельзя делать страницу моно-зеленой. Зеленый должен быть акцентом, не фоном крупных секций.
- Градиенты должны быть мягкими и размытыми, как фоновые SVG DOMA.ai: база `#17313B`, эллипс/свечение `#26C756 -> #4BA2E4`, большой blur. Не делать яркие неоновые пятна.
- Не использовать фиолетовые, синие, бежевые и оранжево-коричневые маркетинговые градиенты.

Desktop шкала:

| Роль | Размер | Line-height | Вес | Цвет |
|---|---:|---:|---:|---|
| Hero H1 | 44px | 48px | 700 | белый |
| Section H2 | 44px | 48px | 700 | белый или темный |
| Muted часть H2 | 44px | 48px | 700 | `rgba(255,255,255,.6)` или `#95A2B0` |
| Wide CTA heading | 31-36px | 34-40px | 700 | белый |
| Testimonial quote | 32px | 38px | 700 | `#222` |
| Product card heading | 32px | 35px | 700 | `#222` |
| Stat/card heading | 22-24px | 26-29px | 700 | белый |
| Body on dark | 16-18px | 22-25px | 400/500 | `#D0D3E5` |
| Button | 16px | 22-24px | 500/600 | зависит от фона |
| Nav/breadcrumb | 14px | 20-21px | 400/500 | белый/серо-синий |

Mobile шкала:

| Роль | Размер | Line-height | Вес |
|---|---:|---:|---:|
| Hero H1 | 28px | 31px | 700 |
| Section H2 | 24px | 26px | 700 |
| Wide CTA heading | 24px | 26px | 700 |
| Card/stat heading | 22px | 24-26px | 700 |
| Body | 14-16px | 20-22px | 400/500 |
| Button | 16px | 22-24px | 600 |

Типографические правила:

- Не использовать отрицательный letter-spacing.
- Не делать hero крупнее 44px на desktop и 28px на mobile.
- Заголовки часто состоят из двух смысловых частей: основная белая/темная часть и приглушенная часть.
- Текст должен быть плотным, но не мелким. В карточках лучше 16px/22px, а не длинные абзацы 14px.

### 3.3 Скругления и поверхности

Основные радиусы:

- hero-карточки и большие секционные карточки: 40px;
- крупные контент-карты внутри темных секций: 40px;
- изображения внутри карточек: 24px;
- stat cards: 24px;
- tabs/chips: 20px;
- CTA buttons: 16px;
- blog/product cards: 20-24px;
- cookie/popup, если показывается статично: 5px, но лучше не включать в лендинг.

Тени почти не используются. Интерфейс строится на цветовых слоях и скруглениях, а не на heavy shadow. Исключение: системные cookie/popups, которые не нужно воспроизводить.

## 4. Layout и сетка

### 4.1 Desktop 1440px

Базовый контейнер:

- viewport: 1440px;
- рабочая ширина контента: 1280px;
- левый край контейнера: около 73px;
- внутренний padding крупных карточек: 40px;
- базовый gap между колонками/карточками: 20px.

Типовые размеры:

- Header: 86px высота.
- Hero: две колонки внутри 1280px:
  - левая hero-card: примерно 846-847 x 484px;
  - правая photo-card: примерно 413 x 484px;
  - gap: 20px;
  - radius: 40px.
- Hero left-card background: локальный `assets/hero-card-bg.webp` или точный аналог оригинального `Frame_48096411` 847 x 484.
- Две benefit cards: 630 x 240px, gap 20px, radius 40px.
- Wide CTA stripe: 1280 x 160px, radius 40px.
- CTA button в wide stripe: 236 x 80px.
- Hero CTA `Подключить`: 140 x 56px, radius 16px.
- Main content card: 1280 x 332-395px, radius 40px.
- Image inside content card: около 586 x 284-347px, radius 24px.
- Stats row: 4 карточки по 305 x 111px, gap 20px, radius 24px.

### 4.2 Mobile 390px

Мобильная версия не должна быть просто сжатым desktop. Оригинал переводит блоки в вертикальный поток:

- внешний горизонтальный padding: 20-24px;
- контент внутри карточек: 24-28px;
- кнопки часто становятся шириной 288-296px;
- hero-card и photo-card идут друг под другом;
- benefit cards и stats складываются в одну колонку;
- H1: 28px/31px;
- H2: 24px/26px;
- кнопки: 54-56px высота, radius 16px;
- большие белые секции сохраняют крупное скругление сверху/снизу.

## 5. Каноническая структура лендинга

Новый статичный лендинг DOMA.ai должен собираться из следующих блоков. Не все блоки обязательны, но порядок лучше сохранять.

### 5.1 Header

Состав:

- логотип DOMA.ai слева;
- nav: Услуги, Умный дом, Приложение жителя, Блог, Партнерам, Работа в Домах;
- телефон;
- outline CTA справа.

Desktop:

- фон `#012025`;
- высота 86px;
- nav 14px/21px, 500;
- outline CTA 58px высотой, radius 16px, border `#707695`.

Mobile:

- логотип слева;
- burger справа;
- меню можно не раскрывать в статичном лендинге, достаточно показать закрытое состояние.

### 5.2 Hero

Структура:

- темный фон страницы;
- внутри контейнера две карточки;
- слева декоративный dark-gradient/card image с текстом;
- справа фотография человека, дома, документов или предметного контекста продукта;
- breadcrumbs над H1;
- H1;
- subhead;
- зеленая CTA.

Не добавлять отдельный тег/бейдж между breadcrumbs и H1: в шаблоне DOMA.ai такого элемента нет, первый экран должен идти сразу от крошек к заголовку.

Формула H1:

```text
[Название продуктовой категории] для [управляющих компаний и ТСЖ]
```

Допустимые точные короткие референсы:

- "Расчетный центр для управляющих компаний и ТСЖ"
- "Страхование ответственности УК и ТСЖ"
- "Аварийно-диспетчерская служба для управляющих компаний и ТСЖ"

Формула subhead:

```text
[Делаем операционную работу] + [для кого/по чему] + [дешевле/надежнее/без найма/без лишних затрат]
```

CTA в hero:

- основной: "Подключить";
- если продукт пробный или платформенный: "Попробовать бесплатно";
- если продукт требует расчета: дополнительный CTA ниже в stripe: "Рассчитать стоимость".

### 5.3 Две benefit cards

Сразу после hero идут две карточки на темном фоне.

Структура каждой карточки:

- иконка/малый SVG вверху слева;
- короткий benefit-heading;
- 1 абзац прикладного объяснения.

Иконки в benefit cards должны быть плоскими line/SVG-иконками с зеленым stroke, без фоновой плашки, бордера и скругленного контейнера. Не сжимать SVG: фиксировать одинаковые `width`, `height` и `flex-basis`.

Формулы заголовков:

- "Снижаем расходы до X%"
- "Улучшаем отношения с жителями"
- "Индивидуальный подход"
- "Ограждаем от лишних расходов"
- "Повышаем финансовую безопасность"

Копирайтинг:

- заголовок должен звучать как результат, не как функция;
- абзац объясняет механизм: что именно берет на себя DOMA.ai.

### 5.4 Wide CTA / стоимость

Темная широкая карточка 1280 x 160px с зеленоватым световым акцентом.

Структура:

- слева крупный текст 31-36px/34-40px;
- справа белая кнопка 236 x 80px;
- radius 40px.

Назначение:

- увести в расчет стоимости;
- заменить интерактивный калькулятор статичным входом.

Для статичного лендинга:

- не нужно реализовывать калькулятор;
- можно показать компактную статичную карточку "Стоимость рассчитывается индивидуально" или оставить только CTA stripe;
- если нужна форма, показать ее как финальный статичный блок без логики.

### 5.5 Главный объясняющий блок с табами

Паттерн:

- centered H2 44px/48px;
- часть H2 приглушена;
- под ним табы/chips 70px высотой;
- active tab зеленый `#00B538`;
- inactive tabs `#17313B`;
- ниже большая карточка с фото слева и списком тезисов справа.

Для статичного лендинга:

- выбрать один активный таб по умолчанию;
- остальные показать как неактивные chips без переключения;
- контент показывать только для активного таба.

Типовые сегменты:

- ТСЖ;
- Управляющим компаниям;
- Крупным холдингам;
- Что включено;
- Почему это выгодно.

### 5.6 Stats row

Ряд из 3-4 темных карточек после main content card.

Формула:

```text
[крупная цифра/квантификатор]
[короткая подпись]
```

Примеры формата:

- 24/7 + действие;
- "От 1 дня" + запуск;
- "7 дней" + внедрение;
- "100%" + отвеченных обращений;
- "По всей России" + география.

Правила:

- цифры не должны быть выдуманными;
- если нет проверенного числа, использовать качественный квантификатор: "по всей России", "любое количество", "индивидуально".

### 5.7 Feature cluster

Темная секция с заголовком слева и набором pill-фич справа.

Паттерн заголовка:

```text
Мы делаем больше, чем просто [базовая услуга]
```

или

```text
Нечто большее, чем просто [базовая операция]
```

Фичи оформлять как маленькие темные pills с белым текстом:

- "Регистрация заявок";
- "Ежемесячная отчетность";
- "Консультации по начислениям";
- "Загрузка квитанций в ГИС ЖКХ";
- "Компенсация судебных расходов".

Не перегружать: 7-10 pills достаточно.

### 5.8 Proof / testimonials

Белая секция с крупным скруглением сверху.

Структура:

- H2: черный + приглушенная часть;
- крупная цитата/результат;
- человек/роль/организация;
- зеленая CTA;
- ниже 3 маленькие карточки отзывов.

Для статичного лендинга:

- если нет реальных отзывов, не выдумывать имена и цитаты;
- заменить блок на "результаты/что получает УК" с фактами и нейтральными карточками;
- реальные цифры и отзывы использовать только из подтвержденных материалов.

### 5.9 Ecosystem cross-sell

Белая секция "все в одном месте".

Структура:

- centered H2;
- 3 карточки:
  - большая: Платформа;
  - малая: Приложение жителя;
  - малая: Сотни услуг для любых задач;
- зеленые CTA внутри карточек;
- ниже можно показать 4 визуальных карточки возможностей платформы.

Этот блок нужен почти всегда: он связывает новый продукт с экосистемой DOMA.ai.

### 5.10 Journal block

Темная секция после белой ecosystem.

Структура:

- H2 "Наш журнал о ЖКХ";
- small green link/action;
- category chips;
- 3 карточки статей;
- карточки: изображение сверху, темная нижняя часть, маленький category pill.

Для генерации:

- можно использовать placeholder-статьи из реального журнала только если есть свежие данные;
- если данных нет, оставить универсальные темы ЖКХ без дат и без выдуманных авторов.

### 5.11 Footer

Темный footer на `#012025`.

Состав:

- логотип;
- краткое описание;
- social icons;
- nav columns: Услуги, Продукты, О нас;
- контакты: email, телефон, адрес;
- юридические ссылки.

Визуально footer должен быть спокойным и компактным. Не превращать его в отдельную промо-секцию.

## 7. Image and asset rules

Hero images:

- real people in office/service context;
- homes, ЖК, documents, calculator/finance, insurance/context objects;
- no abstract 3D, no generic SaaS illustrations, no neon gradients.

Image treatment:

- photos are cropped tightly inside rounded cards;
- no heavy blur over primary subject;
- dark hero background should use `assets/hero-card-bg.webp` or the original DOMA.ai hero-gradient card; do not recreate it with arbitrary CSS gradients when exact matching matters;
- hero/card gradients have no visible border overlay; depth comes from dark base and soft green/blue glow;
- CTA stripe should use `assets/stripe-bg.svg` or an equivalent 1280 x 160 SVG: rounded `#17313B` base plus a blurred `#26C756 -> #4BA2E4` ellipse;
- right hero card should make the product category recognizable at first viewport.

Icons:

- small line/SVG icons with green accent;
- no backing plates, borders, rounded icon containers, or distorted SVG proportions in benefit cards;
- checks are green;
- avoid decorative icon overload.

## 8. Copywriting system

### 8.1 Vocabulary

Use:

- УК, ТСЖ, ЖСК, ТСН;
- жители, собственники;
- МКД, ЖКУ, ГИС ЖКХ;
- заявки, квитанции, начисления, лицевые счета;
- диспетчерская, расчетный центр, страховой полис;
- подрядчики, аварийные службы, бухгалтерия;
- расходы, отчетность, сроки, контроль качества.

Avoid:

- пользователи вместо жители, если речь про жильцов;
- клиенты вместо УК/ТСЖ, если можно назвать субъект точнее;
- абстрактные "решения для бизнеса" без привязки к ЖКХ;
- англицизмы, кроме DOMA.ai и технически неизбежных терминов.

### 8.2 Hero formula

```text
H1: [продуктовая категория] для [УК и ТСЖ]
Subhead: [что делаем] + [какую боль снимаем] + [за счет чего дешевле/надежнее]
CTA: Подключить / Рассчитать стоимость / Получить консультацию
```

Хорошая структура subhead:

- действие: "ведем", "отвечаем", "берем на себя", "помогаем";
- предмет: "заявки", "начисления", "квитанции", "риски";
- выгода: "без найма", "дешевле штата", "без лишних расходов", "с контролем".

### 8.3 Benefit formula

```text
[Глагол в 1 лице множественного числа] + [измеримая или понятная выгода]
```

Примеры направлений:

- снижаем расходы;
- улучшаем отношения;
- повышаем безопасность;
- сокращаем ручную работу;
- разгружаем сотрудников;
- помогаем контролировать сроки.

### 8.4 Explanation formula

Абзацы должны отвечать на вопрос "как именно это работает":

```text
Мы берем на себя [операция], чтобы [роль в УК/ТСЖ] могла [важная задача] без [боль].
```

Примерный синтаксис:

- "Не нужно нанимать...";
- "Мы не только..., но и...";
- "Вы получаете...";
- "Жители...";
- "В конце месяца/по итогам работы...".

### 8.5 CTA language

Основные CTA:

- Подключить;
- Рассчитать стоимость;
- Обсудить условия;
- Получить консультацию;
- Попробовать бесплатно;
- Узнать подробнее;
- Посмотреть услуги;
- Начать пользоваться.

Правила:

- CTA должен быть коротким;
- не использовать "Купить", если продукт требует консультации или расчета;
- для сложных услуг лучше "Рассчитать стоимость" или "Получить консультацию";
- повторять CTA 3-5 раз на странице.

## 9. Статичные замены динамических элементов

Калькулятор:

- заменить на wide CTA stripe;
- при необходимости добавить статичную форму-заглушку с полями "Имя", "Телефон", "E-mail";
- не считать цену на клиенте.

Табы:

- показать один активный таб;
- остальные оставить как визуальные chips;
- без JS-переключения.

Слайдер отзывов:

- показать 1 главный отзыв + 3 карточки;
- если отзывов нет, заменить proof cards без персональных цитат.

Форма:

- статичная верстка допустима;
- поля и checkbox можно показать;
- отправку, маски и валидацию не реализовывать.

Cookie и аналитика:

- не включать в статичный лендинг-макет;
- это инфраструктурный слой, не часть дизайн-системы нового сайта.

## 10. Шаблон задания для AI-генератора

Использовать этот шаблон как входной prompt при создании новой страницы.

```text
Сделай статичный адаптивный лендинг DOMA.ai для продукта: [название продукта].

Аудитория: управляющие компании, ТСЖ/ЖСК/ТСН в сфере ЖКХ.
Главная боль аудитории: [1-2 предложения].
Что продукт делает: [операции, которые берет на себя DOMA.ai].
Главная выгода: [снижение расходов / контроль / доверие жителей / скорость / безопасность].
Основной CTA: [Подключить / Рассчитать стоимость / Получить консультацию].

Обязательные правила стиля:
- темный фон #012025;
- крупные темные карточки #17313B с radius 40px;
- зеленый CTA #2BC359;
- шрифт WixMadeforDisplay с явными @font-face; TildaSans/Arial только как fallback;
- H1 desktop 44/48/700, mobile 28/31/700;
- H2 desktop 44/48/700, mobile 24/26/700;
- header nav/phone/header CTA: 14/21/500;
- hero subhead: 18/25/400, color #D0D3E5;
- wide CTA heading for long cost text: 31/34/700; use 36/40/700 only for shorter CTA headlines;
- контейнер desktop 1280px;
- hero: левая dark-card с текстом, правая photo-card;
- hero background: локальный `assets/hero-card-bg.webp`, not a hand-made gradient approximation;
- CTA stripe: локальный `assets/stripe-bg.svg` или точный аналог с мягким зеленовато-синим blur, без декоративного неона;
- не реализовывать динамику, только статичные состояния.

Структура страницы:
1. Header с логотипом, nav, телефоном и outline CTA.
2. Hero с breadcrumbs, H1, subhead, primary CTA и релевантным фото.
3. Две benefit cards.
4. Wide CTA stripe для расчета/консультации.
5. Главный объясняющий блок: H2, tabs/chips, большая карточка с фото и тезисами.
6. Stats row из 3-4 карточек.
7. Feature cluster: "Мы делаем больше, чем просто [базовая услуга]".
8. Proof section на белом фоне. Не выдумывать отзывы.
9. Ecosystem cross-sell: Платформа, Приложение жителя, Сотни услуг.
10. Journal block.
11. Footer.

Копирайтинг:
- писать на русском;
- использовать лексику ЖКХ: УК, ТСЖ, жители, МКД, ЖКУ, заявки, квитанции, начисления, ГИС ЖКХ;
- говорить через практическую пользу, а не рекламные абстракции;
- не выдумывать цифры, отзывы, имена и юридические утверждения.
```

## 11. Контрольный чеклист качества

Перед сдачей нового статичного лендинга проверить:

- Первый экран сразу похож на DOMA.ai: темный фон, rounded hero, зеленый CTA, фото справа.
- H1 описывает продуктовую категорию для УК/ТСЖ, а не абстрактный оффер.
- На странице есть связка: проблема -> выгода -> CTA -> объяснение -> proof -> ecosystem.
- Визуальная плотность соответствует оригиналу: нет пустого "маркетингового воздуха".
- Зеленый используется дозированно, только для действий и активных состояний.
- Все крупные карточки имеют radius 40px, кнопки radius 16px.
- На mobile блоки идут в одну колонку, кнопки не ломаются, текст не пересекается с изображениями.
- Нет реализованной динамики, если задача - статичный лендинг.
- Нет выдуманных отзывов, метрик, клиентов, юридических гарантий.
- Копирайтинг говорит языком ЖКХ и операционной пользы.
